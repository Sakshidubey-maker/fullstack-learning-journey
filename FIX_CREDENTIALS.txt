# Fix GitHub Credential Issues - Switch from proSAKSHI to Sakshidubey-maker

## Problem: Windows is using cached credentials from old account (proSAKSHI)

## Solution: Clear cached credentials and reconfigure

### Step 1: Clear Windows Credential Manager
1. Press Windows Key + R
2. Type: control /name Microsoft.CredentialManager
3. Press Enter
4. Click "Windows Credentials"
5. Find any entries with "git:https://github.com" or "github.com"
6. Click each one and select "Remove"

### Step 2: Update Git configuration
git config --global user.name "Sakshidubey-maker"
git config --global user.email "your-email@example.com"

### Step 3: Clear Git credential cache
git config --global --unset credential.helper
git config --global credential.helper manager

### Step 4: Try pushing again
git push -u origin main

### Step 5: When prompted for credentials:
Username: Sakshidubey-maker
Password: [Use Personal Access Token from new account]

## To generate Personal Access Token:
1. Login to https://github.com/Sakshidubey-maker
2. Go to: Settings > Developer settings > Personal access tokens > Tokens (classic)
3. Click "Generate new token (classic)"
4. Give it a name: "FullStack Project"
5. Select scope: repo (check the box)
6. Click "Generate token"
7. COPY THE TOKEN (you won't see it again!)
8. Use this token as password when pushing

## Alternative: Use SSH instead of HTTPS (Recommended)
git remote set-url origin git@github.com:Sakshidubey-maker/fullstack-learning-journey.git
